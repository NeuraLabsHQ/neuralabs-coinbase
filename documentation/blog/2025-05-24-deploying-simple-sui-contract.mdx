## Introduction to Sui Smart Contracts

Sui blockchain uses the **Move programming language**, originally developed by Facebook (Meta) for the Diem project. Move is a resource-oriented programming language designed for safe and flexible asset management. Unlike Ethereum's account-based model, Sui uses an **object-based data model** where everything is an object with a unique identifier.

### Key Concepts in Sui Move

- **Objects**: The fundamental unit of storage in Sui
- **Ownership**: Objects can be owned by addresses, shared, or immutable
- **Capabilities**: Special objects that grant permissions
- **Transactions**: Operations that modify object state

## Our Test Contract Architecture

Let's examine the simple test contract we deployed at [`mov_contract/test_contract`](https://github.com/neuralabs/neuralabs-sui/tree/main/mov_contract/test_contract):

<CollapsibleCodeBlock
  title="test.move - Simple Test Contract"
  description="A basic smart contract that creates and transfers test objects"
  language="move"
  defaultCollapsed={true}
>
{`module test_contract::test {
    use sui::object::{Self, UID};
    use sui::tx_context::{Self, TxContext};
    use sui::transfer;

    public struct TestObject has key {
        id: UID,
        value: u64
    }

    public entry fun create_test(ctx: &mut TxContext) {
        let test = TestObject {
            id: object::new(ctx),
            value: 42
        };
        transfer::transfer(test, tx_context::sender(ctx));
    }
}`}
</CollapsibleCodeBlock>

### Contract Analysis

**Struct Definition**:
```move
public struct TestObject has key {
    id: UID,
    value: u64
}
```
- `TestObject` has the `key` ability, making it a Sui object that can be owned
- `UID` provides a unique identifier for the object
- `value` is a simple u64 field for demonstration

**Entry Function**:
```move
public entry fun create_test(ctx: &mut TxContext)
```
- `entry` functions can be called directly from transactions
- `TxContext` provides transaction metadata and capabilities
- Creates a new `TestObject` and transfers it to the transaction sender

## Deployment Process

### Prerequisites

Before deploying, ensure you have:

1. **Sui CLI installed** - Follow the [official installation guide](https://docs.sui.io/build/install)
2. **Active Sui account** with testnet funds
3. **Move.toml configured** in your project directory

For detailed setup instructions, refer to our [Smart Contract Deployment Guide](/docs/sui-integration/smart-contracts/deployment).

## Gas Cost Analysis

Understanding gas costs is crucial for efficient contract deployment:

### Cost Breakdown

1. **Storage Cost** (6.9M MIST): Permanent storage of contract bytecode
2. **Computation Cost** (1M MIST): CPU cycles for deployment processing
3. **Storage Rebate** (978K MIST): Refund for deleted storage
4. **Network Fee** (9.8K MIST): Non-refundable protocol fee

### Optimization Tips

- **Minimize Contract Size**: Remove unused imports and functions
- **Batch Operations**: Combine multiple operations in single transactions
- **Use Shared Objects**: Reduce individual object storage costs

## Security Considerations

Even simple contracts should follow security best practices:

### Move Safety Features

- **Resource Safety**: Objects cannot be duplicated or lost
- **Type Safety**: Strong typing prevents many common errors
- **Ownership Model**: Clear ownership semantics prevent unauthorized access

### Best Practices Demonstrated

1. **Proper Error Handling**: Using `assert!` statements
2. **Access Control**: Functions respect caller permissions
3. **Resource Management**: Proper object creation and transfer

## Troubleshooting Common Issues

### Build Failures

**Error**: `Package dependency resolution failed`
```bash
# Solution: Update Sui framework dependency
sui move build --skip-fetch-latest-git-deps false
```

### Insufficient Gas

**Error**: `InsufficientGas`
```bash
# Solution: Increase gas budget
sui client publish --gas-budget 100000000
```

### Module Name Conflicts

**Error**: `Module name already exists`
```bash
# Solution: Use unique module names or namespaces
```
