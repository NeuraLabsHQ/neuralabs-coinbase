# HPC Neura Execution Engine

The HPC Neura Execution Engine is a powerful workflow execution system designed to process complex AI-driven workflows with high performance and reliability. It provides a flexible, scalable platform for executing neural computation graphs integrated with various AI models and blockchain operations.

<Callout type="info">
The HPC Neura Execution Engine powers the core workflow execution capabilities of the NeuraLabs platform, enabling seamless integration of AI models, blockchain operations, and custom logic.
</Callout>

## Architecture Overview

The HPC Neura Execution Engine is built on a modular architecture that separates concerns and enables scalability:

<Card>
  <CardHeader>
    <CardTitle>Core Components</CardTitle>
  </CardHeader>
  <div style={{ padding: '1rem' }}>
    <ul>
      <li><strong>Flow Executor</strong>: Core engine that manages workflow execution, dependency resolution, and data flow</li>
      <li><strong>Element Registry</strong>: Extensible system for registering and managing workflow elements</li>
      <li><strong>Service Layer</strong>: Integration layer for external services (AI models, blockchain, APIs)</li>
      <li><strong>Event Streaming</strong>: Real-time event propagation system using WebSockets and SSE</li>
      <li><strong>Security Sandbox</strong>: Isolated execution environment for custom code with resource limits</li>
    </ul>
  </div>
</Card>

## How It Works

The execution engine processes workflows defined as directed acyclic graphs (DAGs) where:

1. **Elements** are the nodes that perform specific operations
2. **Connections** define how data flows between elements
3. **Execution** follows dependency order with automatic backtracking
4. **Events** stream in real-time to provide execution visibility

<CollapsibleCodeBlock
  title="Simple Workflow Example"
  description="A basic workflow that takes user input and generates an AI response"
  language="yaml"
>
{`flow_id: simple_ai_chat
elements:
  chat_input:
    type: chat_input
    element_id: chat_input_123
    name: User Message
    
  llm_text:
    type: llm_text
    element_id: llm_text_456
    name: AI Response Generator
    model: "llama-3.3-70b"
    temperature: 0.7
    max_tokens: 1000
    
  end:
    type: end
    element_id: end_789
    name: Return Response
    
connections:
  - from_id: chat_input_123
    to_id: llm_text_456
    from_output: chat_input
    to_input: prompt
  - from_id: llm_text_456
    to_id: end_789
    from_output: llm_output
    to_input: text_input`}
</CollapsibleCodeBlock>

## Quick Start Example

Here's a complete example of a workflow that combines user input, AI processing, and blockchain interaction:

<CollapsibleCodeBlock
  title="AI-Powered Token Transfer Workflow"
  description="A workflow that uses AI to parse user intent and build a blockchain transaction"
  language="yaml"
  defaultCollapsed={false}
>
{`flow_id: ai_token_transfer
flow_definition:
  elements:
    # Input elements
    chat_input:
      type: chat_input
      element_id: chat_input_001
      name: User Request
      description: Captures user's transfer request
      
    # AI processing
    intent_parser:
      type: llm_structured
      element_id: llm_structured_001
      name: Parse Transfer Intent
      model: "llama-3.3-70b"
      temperature: 0.3
      output_schema:
        action:
          type: string
          description: Type of action (transfer, swap, etc)
        recipient:
          type: string
          description: Recipient address
        amount:
          type: string
          description: Amount to transfer
        token:
          type: string
          description: Token type
      wrapper_prompt: |
        Extract the transfer details from the user's message.
        Parse the recipient address, amount, and token type.
        
    # Blockchain interaction
    build_tx:
      type: build_transaction_json
      element_id: build_tx_001
      name: Build SUI Transaction
      node_url: "https://fullnode.mainnet.sui.io"
      contract_address: "0x2::coin::transfer"
      function_name: "transfer"
      function_args:
        - recipient
        - amount
        - token
        
    # Output
    end:
      type: end
      element_id: end_001
      name: Return Results
      
  connections:
    - from_id: chat_input_001
      to_id: llm_structured_001
      from_output: chat_input
      to_input: prompt
    - from_id: llm_structured_001
      to_id: build_tx_001
      from_output: recipient
      to_input: recipient
    - from_id: llm_structured_001
      to_id: build_tx_001
      from_output: amount
      to_input: amount
    - from_id: llm_structured_001
      to_id: end_001
      from_output: action
      to_input: text_input
    - from_id: build_tx_001
      to_id: end_001
      from_output: transaction_json
      to_input: proposed_transaction`}
</CollapsibleCodeBlock>
